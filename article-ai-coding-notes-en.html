Notes on building with AI

AI makes it easy to add features. That makes product judgment more important, not less.

Stop before adding every useful idea

Archive views, tutorials, language switching, mobile builds, and action history all sounded useful. But the app became better when the first experience stayed focused: write, connect, and start one action.

Words are product design

Small wording changes mattered. “Past event” was clearer than “action” for the first column. “Start action” was clearer than a motivational phrase. AI can write code quickly, but the user’s discomfort with a label is valuable evidence.

Ad space

Check regressions by using the app

Fast changes can break existing behavior. Deleting one card should not delete connected cards. A “write next” button should actually move focus. These details only become obvious when the app is used like a real user would use it.

Lesson

AI can speed up implementation. The product still needs a human-held direction.
